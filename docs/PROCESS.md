# 로봇 개발·적용 표준 프로세스

부서 로봇 자동화 과제(인프라 현장 수작업 대체)의 표준 개발 단계와 단계별 보고·에러분석 체계를 정의한다.
이 문서의 단계 구조가 웹 대시보드의 템플릿 선택 기준이며, 판정 잣대의 세부는 [CRITERIA.md](CRITERIA.md), 기록 형식은 [RECORD_SCHEMA.md](RECORD_SCHEMA.md), 각 지표의 원리·산식·적용법은 [METRICS.md](METRICS.md)를 따른다.

## 1. 단계 체인

```
개발 (POC → Pilot) → 개발 (양산 시범 평가) → 인증 (이관심의 · 투자심의) → 양산 적용 (확산 → 운영/관제)
```

- **기간 단계** (데이터가 축적됨): POC, Pilot, 양산 시범 평가, 확산, 운영/관제 → 웹 템플릿 적용
- **심의 게이트** (시점 이벤트): 이관심의(운영승인), 투자심의 → 체크리스트로 관리 (새 데이터를 만들지 않고 기존 증거를 심사)

### 실증 사다리 (무고장 런 — 동일 컴포넌트, 파라미터만 상향)

| 단계 | 무고장 런 | 잣대 | 환경 |
|---|---|---|---|
| POC | 72시간 | 무에러 | 사외 (랩) + 비정상상황 평가 병행 |
| Pilot | 무정지 300시간 | 무정지 | 사내 입고 (공정 연결 없이), 설계 동결 후 |
| 양산 시범 평가 | 무정지 1개월 | 무정지 · 관련 고장만 | 사내 (공정 연결), 판정 합의제 |
| 확산 | 라인별 축약 런 | 무정지 (사전 확정) | 각 적용 라인 (호기별 SAT) |

## 2. 단계별 세부 스텝 · 산출물 · 게이트

### 2.1 POC — "측정의 기준을 만드는 단계"
| 스텝 | 내용 · 산출물 |
|---|---|
| P1 타당성 분석 | 목표, 예상 투자비, 예상 ROI 사전 산출 (비기술 리스크 조기 차단) |
| P2 평가 항목/방법 정의 | FMEA 초판(고장모드 분류 = 이후 전 단계 기록 어휘), 판정기준서 v1 |
| P3 안전인증 컨셉 미팅 | 위험원 식별 → 설계 반영 목록 |
| P4 기성능 평가/검증 | 스펙 합부, 이슈 로그(FRACAS-lite) 개시 |
| P5 자주 시험 | **산출물: 자주 시험 결과서 (사외 72시간 무에러 + 비정상상황 평가)** |

**게이트**: 기성능 스펙 충족 + 72h 무에러 통과 + 비정상 시나리오 합부 + 상위 심각도 미해결 0건 + FMEA 상위 리스크 조치계획.

### 2.2 Pilot — "고쳐가며 성장시키는 단계"
| 스텝 | 내용 · 산출물 |
|---|---|
| L1 가동 지표 정의 | MCBF/MTTR/가동률 산식 확정 (양산 시범 평가와 동일 산식) |
| L2 신뢰성 평가 (TAAF) | 반복운전→고장→FRACAS→재운전, MCBF 성장 추적 |
| L3 임시 사용 안전 승인 | 사내 반입·가동 근거 |
| L4 퀄 항목/사양 정리 | 퀄 잣대를 사용 전 단계에서 확정 |
| L5 실증 런 | **사내 입고(공정 연결 없이) 무정지 300시간** |
| 산출물 | 신뢰성 시험 사양서/결과서, 하드웨어 품질표준사양서, 안전인증서 |

**게이트**: 300h 무정지 통과 + 만성(재발) 고장 0 + 시정조치 검증 마감 + 안전인증서.

### 2.3 양산 시범 평가 — "합의된 잣대로 실증하는 단계"
| 스텝 | 내용 · 산출물 |
|---|---|
| M1 개발/수혜 부서 협의 | 성공 기준을 계약 파라미터로 동결 (config 고정, 사후 변경 불가) |
| M2 양산화 검토 · 시범라인 선정/운영 | 실부하 운영, **사내(공정 연결) 무정지 1개월** |
| M3 운영 안정성 검증 | MTTR 실측, 가동률 산출 |
| M4 안전 검수 · 위험성 평가 · 가동인증 | 설치 상태 기준 |
| M5 이관 심의 산출물 준비 | 퀄 결과 보고서, 안전 매뉴얼·SOP, 안전인증서 |

**게이트**: 무정지 1개월(관련 고장 ≤ 한도) + MTTR/가동률 목표 + 안전 3종 + 이관 산출물 완비.

### 2.4 인증 — 심의 게이트 2종 (기간이 아니라 이벤트)
- **이관심의(운영승인, ORR)**: 기능/성능 검증·퀄 확정, 매뉴얼, 표준관리·자산등록, 점검표, 이관 산출물 점검/합의. 미종결 FRACAS는 건별로 종결/조건부 이관(carry-over)/위험수용(waiver) 처분 → **Known Issues Register 단일본** (기한·책임자 서명).
- **투자심의**: 라인 생산 영향성 평가(제조·기술혁신 공동), 확산 도입 투자결정. 실증치를 동결 스냅샷으로 사용, 보수치(MCBF 신뢰구간 하한)로도 경제성 성립을 입증.

### 2.5 양산 적용 — 확산 모드 / 운영·관제 모드
- **확산**: 기준 구성 동결(승인 편차만 예외), 호기·라인별 양산 퀄(설치→SAT→축약 무고장 런), 가동지표 검증, 횡전개 수립. 고장은 원인계층(설계/제작·조립/설치/운영)으로 분류 — 설계성 고장은 전 함대 리스크로 즉시 에스컬레이션.
- **운영/관제**: 통합관제·모니터링(알람 자동수집), 비상상황 대응(에스컬레이션 매트릭스·훈련), 유지보수(PM→CBM)·CIP. 월간 RAM 보고. 필드 고장모드 → 차기 과제 FMEA 환류.

## 3. 단계별 에러분석 · FRACAS 운용 (설득력의 핵심)

단계가 오를수록 분석이 답하는 질문이 바뀐다. 그 단계의 질문에 맞는 종류의 증거를 내는 것이 설득력이다.

| 단계 | 분석의 질문 | FRACAS 운용 | 분석 화면의 주인공 |
|---|---|---|---|
| POC | 컨셉의 병인가, 고칠 수 있는 병인가 | FRACAS-lite (필수 5필드), 통계 금지 | 전수 4분류 보드 (컨셉리스크 0 입증) |
| Pilot | 우리는 수렴하고 있는가 | full FRACAS (버전·무발생검증 필수) | MCBF 성장곡선 + Pareto + 재발 0 |
| 양산 시범 | 잔여 고장률이 계약 기준 이내인가 | FRACAS 지속 + 관련/비관련 합동판정 | 판정 대장 (증거 첨부) |
| 인증 | 남은 결점을 어떤 조건으로 안고 가는가 | 열린 건 처분 (carry-over/waiver) | Known Issues Register |
| 확산 | 설계 고장인가, 이 호기만의 문제인가 | 원인계층 분류 + 호기별 층화 | 함대 RAM + 원인계층 차트 |
| 운영 | 어떤 고장부터 없애는 게 경제적인가 | 필드 FRACAS (알람 승격 기준) + 8D | 다운타임 비용 Pareto + CIP |

## 4. 3원칙 (전 단계 공통)

1. **고장 레코드 스키마는 전 과제·전 단계 공통** — POC부터 같은 형식으로 기록해야 단계 간·과제 간 이력이 이어진다 ([RECORD_SCHEMA.md](RECORD_SCHEMA.md))
2. **보고 템플릿은 단계별 소수 종** — 개발(POC/Pilot 모드) · 실증(양산 시범) · 운영(확산/운영 모드), 심의는 체크리스트
3. **판정 잣대·리셋 규칙은 데이터 이전에 확정** — 사후 변경·재해석 금지 ([CRITERIA.md](CRITERIA.md))

## 5. 재정비 권고 (반영 여부는 부서 결정)

1. 인증 단계의 두 심의는 안건·서명 주체 분리 (이관심의=운영조직 인수 서명 → 그 동결 실증치로 투자심의=투자위 의결)
2. "무정지 MTBI 300시간"의 용어 정리 — MTBI(평균 간격 지표)와 "연속 무정지 300h 달성"(실증 런)은 다른 개념. CRITERIA.md의 조작적 정의를 사용
3. FRACAS 오너십 RACI 사전 확정 — 단계×원인계층별 폐루프(원인→시정→검증) 책임자. 이관 순간의 오너십 공백이 최대 실패점
4. 무정지 잣대에 제품 품질 지표 병기 — "무정지인데 불량 양산" 방지
5. 설계 동결 후 변경관리 — 변경 등급 × 재검증(축약 런 재수행) 매트릭스 사전 확정
6. 법정 인증(KC·산업안전보건법)은 사내 인증과 별도 트랙으로 표기
